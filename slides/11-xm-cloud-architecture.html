Component Architecture

  • JSS (JavaScript Services) SDK for React/Next.js
  • Each component maps to a Sitecore rendering
  • Data sources define content schema per component
  • Placeholder-based layout composition
  • Shared component library across multi-site
  • Design tokens enforced via Tailwind / CSS vars
💻

Next.js Rendering Host

  • SSG + ISR for marketing pages (fast TTFB)
  • SSR for personalized / authenticated pages
  • API routes as middleware for OC + integrations
  • Deployed on Vercel or any Node.js host
  • Preview mode for editors (live content preview)
  • Automatic revalidation on publish events

Layout Service

  • REST or GraphQL endpoints
  • Returns page layout as JSON tree
  • Component hierarchy + field data
  • Language-aware responses

Experience Edge

  • Global CDN for layout & content
  • Auto-publish from XM Cloud
  • GraphQL API at the edge
  • Sub-50ms content delivery worldwide

Headless SXA

  • Pre-built headless components (Hero, Promo, Navigation)
  • Grid & column layout components
  • Theme and variant system
  • Extensible via custom renderings
